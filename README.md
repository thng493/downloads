## 🔒 Security and verification

We prioritize your security. Every release undergoes a checksum verification process. This process confirms that the file on our server matches the original version. It ensures that the files remain free from corruption or changes during hosting.

To verify a file manually:

1.  Open the Command Prompt on your computer.
2.  Type `certutil -hashfile [filename] SHA256`. 
3.  Replace `[filename]` with the actual name of your file.
4.  Compare the results with the hash values listed in our release notes.

## ⚙️ Configuration settings

The vxcloud Desktop app includes a Settings menu. Click the gear icon to adjust your preferences. You can set the default region for your cloud services, change your update frequency, and toggle interface themes. Changes save automatically when you close the settings window.

## 📋 Using the browser extension

The browser extension needs permission to access cloud data. Install it from your browser store or the ZIP file provided here. Once you install it, look for the vxcloud icon in your browser toolbar. Click the icon to sign in if your specific cloud setup requires account authentication for personal data storage. Settings within the extension allow you to pin the icon for quick access. 

## 🌐 Connecting to cloud services

Use the vxcli tool to link your local machine to your cloud environment. Open your terminal and type your connection string. This binds your local folder to your remote cloud workspace. You can then move, copy, and edit files as if they existed on your desktop. Our tools handle the encryption during transit. This keeps your data private while it moves between your computer and the server.
